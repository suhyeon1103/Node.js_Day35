9. **Node.js에서 pipelining을 사용하는 주된 이유는 무엇인가요?**
-> A
-> 성능향상: pipelining을 사용하면 여러 작업을 동시에 처리할 수 있어, I/O 작업의 대기 시간을 줄이고 전반적인 성능을 향상시킬 수 있습니다.
-> 비동기 처리: Node.js의 비동기 특성을 활용하여, 작업이 완료될 때까지 기다리지 않고 다음작업을 수행할 수 있어 효율적인 자원 활용이 가능합니다.
-> 코드 가독성: 파이프라인을 사용하면 데이터 흐름을 명확하게 표현할 수 있어 코드의 가독성이 높아지고 유지보수가 용이해집니다.
-> 에러 처리: 각 단계에서 발생할 수 있는 에러를 중앙 집중식으로 관리할 수 있어, 에러처리가 간편해집니다.
-> 스트림 처리: 대량의 데이터를 효율적으로 처리하기 위해 스트림과 함께 사용하면 메모리 사용량을 줄이고, 데이터 처리 속도를 높일 수 있습니다.

   C) 메모리 사용량을 늘리기 위해.
   D) 보안성을 강화하기 위해.
